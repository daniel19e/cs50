sort1 uses: bubble sort

How do you know?: because for worst case scenario it runs slow like selection sort but when the array is already sorted(best case scenario) it runs way faster.

sort2 uses: merge sort

How do you know?: Because the best case scenario and worst case scenario take the same time to run, and it's way quicker than the other 2

sort3 uses: selection sort

How do you know?: running time is pretty slow but it's the same for both worst and best scenario, which indicates it's selection sort.
